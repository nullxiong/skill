# 行程工作流，报销

## 目标

自动化完成定期差旅报销前置工作：

1. 从 QQ 邮箱检索指定时间段内的发票邮件。
2. 下载高铁、住宿、网约车相关发票和行程单到本地目录。
3. 避免重复下载，尤其是同主题、同附件名的滴滴邮件。
4. 解析本地发票，梳理差旅行程和费用。
5. 基于报销 Excel 模版另存新报销表，不修改原模版。

## 邮箱配置

使用 QQ 邮箱 IMAP 授权码，不使用浏览器登录 QQ 邮箱。

临时配置文件：

```text
work/.qq_mail_env
```

内容格式：

```bash
QQ_EMAIL=你的QQ邮箱@qq.com
QQ_IMAP_AUTH_CODE=你的QQ邮箱IMAP授权码
```

注意：这里用的是 QQ 邮箱的 IMAP 授权码，不是 QQ 密码。完成后可在 QQ 邮箱后台撤销授权码。

## 发票检索范围

默认检索近一个月邮件。当前这次使用的时间范围是：

```text
2026-05-01 到 2026-06-01
```

如果只检索当天新增发票，例如滴滴当天新邮件：

```text
2026-06-01 到 2026-06-01
```

实际 IMAP 查询使用 `--since 2026-06-01 --before 2026-06-02`。

## 邮件下载规则

### 基础规则

1. 连接 QQ 邮箱 IMAP。
2. 按日期范围检索邮件。
3. 按关键词筛选相关邮件。
4. 下载附件到对应分类目录。
5. 如果同名文件已经存在，默认跳过，不覆盖。

### 关键词

高铁：

```text
高铁、火车、铁路、12306、中国铁路、电子客票
```

住宿：

```text
酒店、住宿、宾馆、华住、携程、同程、去哪儿、结账单
```

网约车：

```text
滴滴、DIDI、曹操、T3、高德、美团打车、网约车、打车、用车、出行
```

### 滴滴邮件特殊规则

滴滴邮件可能出现多封邮件使用同一主题、同一附件名的情况。不能只按附件名跳过，否则会漏下后续邮件里的发票和行程单。

处理方式：

```text
文件名增加“邮件01 / 邮件02 / 邮件03 ...”后缀
```

示例：

```text
20260601_滴滴出行电子发票及行程报销单_邮件05_滴滴电子发票A.pdf
20260601_滴滴出行电子发票及行程报销单_邮件05_滴滴出行行程报销单A.pdf
```

## 发票解析规则

### 高铁

从 ZIP 中提取 PDF，读取：

- 发票号码
- 乘车日期
- 出发站
- 到达站
- 票价

### 住宿

优先使用结账单判断入住信息：

- 酒店名称
- 入住日期
- 离店日期
- 消费合计

住宿电子发票用于确认：

- 发票号码
- 价税合计金额

### 网约车

行程报销单用于确认：

- 行程起止日期
- 上车时间
- 城市
- 起点
- 终点
- 金额

电子发票用于确认：

- 发票号码
- 出行日期
- 金额

## Excel 填写规则

打开模版：

```text
/Users/xxx/work/00-亚信安全/08-报销/差旅报销 -模版.xlsx
```

只填写副本，不修改原模版。

填写工作表：

```text
销售费用报销表
```

主要填写字段：

- 出发地
- 目的地
- 天数
- 起止日期
- 火车费
- 出租车费
- 住宿费
- 出差补助，按 75 元/天
- 发票号
- 事由

不填写：

- 小计
- 累计金额

这些保持 Excel 原公式自动计算。

3. 连接 QQ 邮箱检索对应时间段发票。

10. 核对 Excel 中四类关键金额：火车费、出租车费、住宿费、补助。

## 注意事项

- QQ 邮箱授权码不要写入公开文档或长期保存。
- 滴滴发票必须同时检查电子发票和行程报销单。
- 同一天多封滴滴邮件时，必须用邮件编号后缀区分。
- 住宿费用优先以结账单和电子发票共同核对。
- 退票费如需报销，可计入火车费；如公司不允许，应人工剔除。
- Excel 原模版不要直接修改，只能另存副本。
