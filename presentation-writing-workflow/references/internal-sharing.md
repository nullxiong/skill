# Internal Sharing

Use for internal talks, knowledge sharing, culture/strategy storytelling, technology sharing, or keynote-like sessions where attention and memory matter more than dense detail.

## Strategy

- Lead with a narrative hook, conflict, question, or scene.
- Build a clear story arc: context -> tension -> discovery -> method -> payoff -> call to action.
- Prefer fewer words, stronger page/screen concepts, and memorable chapter transitions.
- Use vivid metaphors only when they clarify the topic.
- Avoid corporate report language unless the user asks for a formal tone.

## PPT Image Draft

- Use cinematic compositions, bold contrast, strong hero visuals, large titles, and dramatic section divider pages.
- Make each page independently memorable.
- Keep text short enough to survive image generation and live presentation.
- Use visual intensity high for opening, chapter dividers, turning points, and closing.

## HTML

- Use immersive scroll storytelling, sticky sections, progressive reveals, and strong transitions.
- Let long-form details appear as expandable blocks or secondary sections.
- Use motion to support story beats, not as decoration.
- Consider generated background images, WebGL/Three.js scenes, or full-bleed visual sections when useful.

## Common Outline Shape

1. Opening hook
2. Why this matters now
3. Core story or problem
4. Key insight or framework
5. Examples or proof
6. What changes next
7. Closing message
