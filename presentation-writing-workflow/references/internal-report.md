# Internal Reporting

Use for leadership updates, project reviews, progress reports, issue escalations, business reviews, or decision meetings.

## Strategy

- Start with the conclusion, then provide evidence.
- Make decisions, risks, blockers, ownership, and next steps explicit.
- Prefer structured sections over dramatic storytelling.
- Use concise titles that state the point, not just the topic.
- Keep visual style controlled, readable, and businesslike.

## PPT Image Draft

- Use executive summary pages, metric cards, milestone timelines, risk matrices, decision pages, and roadmap pages.
- Keep visual intensity low to medium unless the user asks for a more polished keynote style.
- Use charts and tables when they reduce ambiguity.
- Make the page hierarchy obvious: conclusion first, evidence second, detail third.

## HTML

- Use report/dashboard layouts with summary bands, KPI cards, tables, filters if needed, and clear section anchors.
- Prefer scanability over immersion.
- Use restrained animation: loading transitions, chart reveals, section anchors, expandable details.
- Design for repeated reading and comparison.

## Common Outline Shape

1. Executive summary
2. Current status
3. Key progress
4. Metrics/evidence
5. Risks and blockers
6. Decisions needed
7. Next steps and owners
