# Customer Briefing

Use for customer-facing proposals, solution briefings, project reports, executive presentations, product capability briefings, and pre-sales materials.

## Strategy

- Lead with the customer's context and problem, not the vendor's capabilities.
- Build trust through clear logic, proof, cases, architecture, and implementation path.
- Use professional, clean language with moderate technology flavor.
- Avoid excessive hype, visual clutter, and ungrounded claims.
- Make business value and delivery feasibility visible.

## PPT Image Draft

- Use premium proposal visuals: clean spacing, modern typography, structured diagrams, restrained tech backgrounds.
- Use visual intensity medium: polished but not overwhelming.
- Include architecture, process, roadmap, capability map, case proof, and value summary pages where relevant.
- Keep customer-facing claims specific and supportable.

## HTML

- Use an interactive briefing/proposal page with strong navigation, solution sections, architecture diagrams, case cards, implementation timeline, and value summary.
- Use subtle motion and technology cues: fine grid, line work, data traces, glass or light accents only when tasteful.
- Preserve credibility by prioritizing clarity and hierarchy.

## Common Outline Shape

1. Customer context
2. Challenges and goals
3. Proposed solution
4. Core capabilities
5. Architecture or operating model
6. Implementation roadmap
7. Proof/cases
8. Value and next steps
